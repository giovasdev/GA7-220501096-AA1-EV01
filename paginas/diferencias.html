🔄 Versionamiento Local vs Remoto

Comprende la diferencia entre controlar tu código en tu equipo y hacerlo en la nube

Git permite llevar el control de versiones de un proyecto. Sin embargo, es importante diferenciar entre el versionamiento local, que se gestiona en tu máquina, y el versionamiento remoto, que se almacena en servidores como GitHub, GitLab o Bitbucket.

Versionamiento Local

Es el proceso de control de versiones que ocurre en tu equipo. Usas Git para guardar cambios, revisar historial, crear ramas y hacer commits, todo sin conexión a internet.

🔧 Herramientas: Git CLI, VSCode

✅ Ventajas: Rápido, privado, sin conexión

Versionamiento Remoto

Es el almacenamiento y colaboración de tu código en servidores en la nube. Permite trabajar en equipo, hacer pull requests y mantener respaldos accesibles desde cualquier lugar.

🌐 Plataformas: GitHub, GitLab, Bitbucket

✅ Ventajas: Copias seguras, colaboración, automatización
